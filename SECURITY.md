# Security Policy

This repository contains a joke Internet-Draft, associated community
infrastructure, and an EU Declaration of Conformity.

## Reporting a Vulnerability

If you believe you've found a security issue, please report it privately:

- Email: security@propertools.be
- Subject: `[propertools/CVSSv5] Security report`

Please include:

- A clear description of the issue
- Steps to reproduce (or a minimal PoC)
- Impact assessment (what an attacker could do)
- Your emotional reaction upon discovery, for scoring purposes

- Temporal paradoxes arising from misuse (or use) of RFC 9999
- Dave from InfoSec

## Disclosure

We prefer coordinated disclosure. If you contact us first, we will:

- Acknowledge receipt within a reasonable timeframe
- Go for a long walk, sigh repeatedly
- Work to reproduce and remediate
- Make another coffee, read some email to avoid thinking about it
- Score the vulnerability using the very same system defined in this document
- Join the wrong Zoom meeting, hang up, rejoin the wrong Zoom meeting again
- Credit you in the Acknowledgements (if so desired)
- Add you to the Leaderboard if the vulnerability is sufficiently absurd

## Scoring a Vulnerability

We use CVSS v5.0 (this document).

Example severity calculation:

```
CVSS v5 (CVSSv5 Repo Edition):
Base Score: ☠️☠️☠️
Vector: ABSURDITY:HIGH/COHERENCE:MODERATE/RFC-COMPLIANCE:WORRYING
Environmental adjustment: +☠️ if discovered on a Friday
Temporal adjustment: Non-canonical after 19 January 2038
```

Note: vulnerabilities in the scoring system used to score
vulnerabilities are scored using the scoring system, unless
the scoring system is the vulnerability, in which case
scores resolve to zero per Section 2.3 and all accountability
is reassigned to future leadership.
